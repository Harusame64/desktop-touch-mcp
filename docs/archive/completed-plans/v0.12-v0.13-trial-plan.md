# v0.12 / v0.13 新機能 試用プラン

対象バージョン: v0.13.1（現行 npm 公開版）  
目的: Auto Guard / HotTargetCache / SuggestedFix / Timeline / browser_eval(withPerception) / mouse_drag ブロック を実際に動かして動作を目視確認する。

---

## Phase 1 — 準備

- [ ] 1-1. `screenshot(detail='meta')` でデスクトップ状態を確認
- [ ] 1-2. メモ帳（`notepad.exe`）を起動
- [ ] 1-3. Chrome を起動 or 既存ウィンドウを確認
- [ ] 1-4. `get_context` で現在のフォーカスを確認

---

## Phase 2 — Auto Guard（v0.12）

**ねらい**: `windowTitle` を指定するだけでガードが効くことを確認する。

- [ ] 2-1. `mouse_click` に `windowTitle: "メモ帳"` を付けてクリック → レスポンスに `post.perception` が返ることを確認
- [ ] 2-2. **わざとガード失敗**させる: 存在しないウィンドウタイトルを指定 → エラー + `suggestedFix` が返ることを確認
- [ ] 2-3. `DESKTOP_TOUCH_AUTO_GUARD=0` 相当で無効化したときとレスポンスを比較（env は設定しないで差を観察）

---

## Phase 3 — HotTargetCache（v0.12）

**ねらい**: 同じ対象に連続操作してキャッシュが効いていることをレスポンス速度で確認する。

- [ ] 3-1. メモ帳に `keyboard_type` で文字入力（`windowTitle` 付き）→ 1 回目
- [ ] 3-2. 同じ操作を 3〜4 回繰り返す → 2 回目以降が明らかに速いか観察
- [ ] 3-3. `get_history` でキャッシュヒット情報が返るか確認（`recentTargetKeys`）

---

## Phase 4 — SuggestedFix（v0.12 + v0.13）

**ねらい**: ガード失敗時の `fixId` → `mouse_click({ fixId })` で一発復旧する流れを確認する。

- [ ] 4-1. 別のウィンドウをフォーカスした状態でメモ帳に `mouse_click(windowTitle="メモ帳")` → ガード失敗を誘発
- [ ] 4-2. レスポンスから `fixId` を取得
- [ ] 4-3. `mouse_click({ fixId })` で復旧 → 成功することを確認
- [ ] 4-4. v0.13 追加分: `keyboard_type({ fixId })` / `click_element({ fixId })` でも同様に復旧できることを確認

---

## Phase 5 — Target-Identity Timeline（v0.13）

**ねらい**: 操作対象ごとの履歴が蓄積・参照できることを確認する。

- [ ] 5-1. Phase 2〜4 の操作後に `get_history` → `recentTargetKeys` が表示されることを確認
- [ ] 5-2. `perception_read` で `recentEvents` を取得 → タイムライン内容を目視確認
- [ ] 5-3. `perception://target/{key}/timeline` リソースを参照（MCP resource URI）

---

## Phase 6 — browser_eval withPerception（v0.13）

**ねらい**: `withPerception: true` で DOM 状態も一緒に返ってくることを確認する。

- [ ] 6-1. Chrome で任意のページを開く（例: `about:blank` or `google.com`）
- [ ] 6-2. `browser_eval` で JS を実行（例: `document.title`）、`withPerception: false`（デフォルト）
- [ ] 6-3. 同じ JS を `withPerception: true` で実行 → レスポンスに perception 情報が追加されていることを確認
- [ ] 6-4. 循環参照が発生しやすい JS オブジェクト（`window` など）を渡してもクラッシュしないことを確認

---

## Phase 7 — mouse_drag クロスウィンドウブロック（v0.13）

**ねらい**: 別ウィンドウをまたぐドラッグがデフォルトでブロックされることを確認する。

- [ ] 7-1. メモ帳 → Chrome をまたぐ座標で `mouse_drag` を実行 → エラー/ブロックが返ることを確認
- [ ] 7-2. `allowCrossWindowDrag: true` を付けて同じ操作 → 実行が通ることを確認
- [ ] 7-3. 同一ウィンドウ内ドラッグ（メモ帳内テキスト選択）は `allowCrossWindowDrag` なしで動くことを確認

---

## 進め方

- Claude Code セッション内で MCP ツールを直接呼び出して試す
- 各 Phase のチェックボックスを `[x]` に flip しながら進める
