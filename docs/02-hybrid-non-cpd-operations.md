# 非CDP操作のハイブリッド化（UIA + Vision/OCR + Rust）実装計画

## 1. 背景と課題 (Problem)
現在、`desktop-touch-mcp` はRust製の高速UIAエンジンを搭載しているが、ゲーム、RDP、独自レンダラ（非アクセシビリティモードのElectron等）ではUIAツリーが空、または巨大な単一の `Pane` になってしまう。
現状の代替手段であるWindows内蔵OCRへの単純なフォールバックは、アンチエイリアスや低解像度に弱く、UI文字の誤認識（「僅い蓍手」など）が多発しており、LLMの操作精度を落とす原因となっている。

## 2. 最先端リサーチ：GUIエージェントのSoTAアーキテクチャ
最新のLLMベースのGUIエージェントのトレンドを調査した結果、Microsoftの「OmniParser」などの純粋なビジョンベースのスクリーン構文解析がSoTA（最高性能）とされている。
* **Set-of-Mark (SoM) アプローチ**: OmniParserは、スクリーンショット内の操作可能な領域（アイコンやテキスト）を検出し、その機能的セマンティクスを抽出して、構造化されたDOMのような表現を生成する。
* **LLMの負荷軽減**: 解析フェーズで画面の意味情報を事前に抽出することで、LLM（GPT-4VやClaudeなど）が画面解析にかける推論負荷を軽減し、アクション予測に集中させることができる。

この「画面上の要素にバウンディングボックスと番号ID（`[1]`, `[2]`など）を直接描画してLLMに見せる」手法を取り入れることで、OCRの微細な文字化けにLLMが引きずられるのを防ぎ、ID指定による確実なクリックを実現できる。

## 3. 実装方針：段階的フォールバックとRustのフル活用 (Solution)
既存の「Rust UIAエンジン」と「Rust SIMD画像処理エンジン」をシームレスに結合し、SoMパイプラインを構築する。画像処理ライブラリ（`sharp`）への依存を減らし、Rustに「LLMの視覚野」としての新たな大役を任せる。

### Step 1: UIAのSparsity（過疎度）判定
1. 従来通りRustエンジンで `FindAllBuildCache` を実行。
2. 取得できた要素数が極端に少ない（例: `elements.length < 5`）、または画面全体の90%以上を占める `Pane` が1つだけの場合、「UIA-Blind（視界不良）状態」と判定し、Step 2へ移行。

### Step 2: Rustエンジンによる画像前処理（OCR精度向上）
1. RustのSIMD拡張を活かし、取得したスクリーンショットの対象領域をメモリ上で直接 **2倍〜3倍にアップスケール ＋ グレースケール化 ＋ コントラスト強調** する。
2. これにより、ClearType等のアンチエイリアスによる「文字の境界のぼやけ」を物理的に消し飛ばし、後続の `Windows.Media.Ocr` に渡した際の認識精度を劇的に向上させる。

### Step 3: OCR実行と要素のクラスタリング
1. 前処理された画像を C# 側のOCRブリッジ（`win-ocr.exe`）に渡し、テキストと座標を取得。
2. 近接する単語ブロックを結合し、クリック可能な「要素（ボタンやリンクの塊）」としてグルーピングする。

### Step 4: Rustエンジンによる Set-of-Mark (SoM) 画像の高速レンダリング
1. **【Rustの最大の見せ場】** 検出された各要素の座標に対し、Rust側で直接画像バッファを操作し、**「赤いバウンディングボックス」と「連番のIDバッジ（例: [ 1 ]）」を描画**する。
2. TS側の `sharp` を経由せず、Rustのベクタ/ピクセル描画で完結させることで、数ミリ秒でSoM画像を生成する（バックログの「画像処理の完全Rust化」の第一歩）。

### Step 5: LLMへの統合フォーマット (Output)
ツール（`screenshot(detail="text")`のフォールバック時など）のレスポンスとして、以下を返却する。
* **画像**: Rustがレンダリングしたバウンディングボックス＋ID付きのSoM画像。
* **テキスト**: `[{"id": 1, "text": "検索", "clickAt": {"x": 150, "y": 200}}, ...]` というIDマッピングのJSON。

これにより、LLMは「ID: 1の場所をクリックしたい」と考え、そのまま `click_element(id="1")` または `mouse_click` を叩くだけの超安定ワークフローが完成する。
